# Onboarding Flow Designs

Need to onboard the following entities

1. Masjid Onboarding
   - What
     1. There are two types of onboarding
        - Partial Listing: Any InMasjid user can partially add masjid listing with important details such as
          + Name
          + Location
          + Photo
          + Salah Timings
          - Imaam/Admin Number
          - Optional details
        - Verified Listing: InMasjid Onboarding Team (on-field) role should
          1. Verify the listing on-field
          2. Speak with 3 person quorum: Imaam, Muadhin and Admin and make them Super masjid Admin
          3. Teach them the super admin role and onboard them
   - Who
     1. InMasjid Onboarding Team role: Need proper onfield training
     2. Any Musallee can 
        - request to add a partial Masjid listing.
        - A log is always maintained for all the activities and sadaqah e jaariyah
   - How
     1. Submit all the required information
     2. Fill Optionals
     3. Onboard masjid super admins with training
     4. Need a huge help library and immediate support helpline
     5. How to retrain? Keep the system so easy that the retain is just not required
     6. Maintain an LMS to keep the super admins up to date

2. Musallees
    - How
      1. OTP (mPin for masjid super admin changes)
    - What
      1. Guest (any user not part of the InMasjid ecosystem)
      2. Musallee
      3. Super Musallee
      4. Masjid Team
      5. InMasjid Team
         - Groups and Permissions
